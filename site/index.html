Rust + axum + tokio + MongoDB

작고 빠른 URL 단축 API

rlnk는 짧은 링크 생성, 리다이렉트, 삭제, 통계 조회를 단순한 API로 처리하는 고성능 URL 단축 서비스입니다.

Docker Compose로 실행 API 보기
  • HTTP/HTTPS URL 검증
  • Authorization: Bearer APP_KEY
  • TTL: 10m, 24h, 7d
  • ACCESS_CACHE_SIZE 기본값 1024

프로젝트 개요

API는 작게, 실행은 간단하게

Rust 비동기 HTTP 서버와 MongoDB 저장소를 바탕으로 명시적인 오류 처리와 작은 컨테이너 이미지를 제공합니다. 설치와 API 사용 흐름은 필요한 단계만 남겨 짧게 유지했습니다.

비동기 Rust 서버

axum과 tokio 기반 비동기 서버입니다. MongoDB 접근도 async API로 처리해 핸들러 안에 blocking I/O를 두지 않습니다.

TTL과 최근 접근 캐시

10m, 24h, 7d 같은 duration 형식으로 만료 시각을 계산하고, 반복 조회는 크기 제한 캐시로 DB 접근을 줄입니다.

관리 API 보호

링크 생성, 삭제, 통계 조회는 Authorization: Bearer <APP_KEY> 헤더로 보호합니다.

API

생성부터 통계까지 엔드포인트 4개

사용자 지정 hash는 받지 않고 서버가 URL-safe hash를 만듭니다. 각 항목을 펼치면 바로 실행해 볼 수 있는 curl 예제가 나옵니다.

POST /gen 인증 필요 원본 URL과 선택 TTL을 받아 짧은 URL과 hash를 반환합니다.
curl 링크 생성 
curl -X POST https://rlnk.example.com/gen \
  -H 'Authorization: Bearer change-me' \
  -H 'Content-Type: application/json' \
  -d '{"url":"https://example.com","ttl":"10m"}'
GET /{hash} 인증 없음 원본 URL로 리다이렉트합니다. 만료됐거나 없는 hash는 404입니다.
curl 리다이렉트 
curl -i https://rlnk.example.com/abc123
DELETE /{hash} 인증 필요 지정한 hash의 링크를 삭제합니다. 없는 hash는 404입니다.
curl 링크 삭제 
curl -X DELETE https://rlnk.example.com/abc123 \
  -H 'Authorization: Bearer change-me'
GET /stat 인증 필요 원본 URL, 접근 횟수, 생성·만료·마지막 접근 시각을 반환합니다.
curl 통계 조회 
curl https://rlnk.example.com/stat \
  -H 'Authorization: Bearer change-me'

설치

Docker Compose로 바로 실행

.env와 docker-compose.yml 예제를 참고해 배포하세요. APP_HOSTNAME에는 실제 공개 주소를 넣습니다.

1

환경 파일 준비

.env에는 앱 키와 공개 호스트명처럼 환경마다 달라지는 값만 넣습니다.

2

스택 시작

docker compose up -d로 rlnk 컨테이너를 백그라운드에 띄웁니다.

.env 
APP_KEY=change-me
APP_HOSTNAME=https://rlnk.example.com
APP_BIND_ADDR=0.0.0.0:8080
MONGO_URI=mongodb://localhost:27017
MONGO_DATABASE=rlnk
MONGO_COLLECTION=links
HASH_LENGTH=8
ACCESS_CACHE_SIZE=1024
docker-compose.yml 
services:
  rlnk:
    image: ghcr.io/suapapa/rlnk:latest
    env_file:
      - .env
    ports:
      - 8080:8080
